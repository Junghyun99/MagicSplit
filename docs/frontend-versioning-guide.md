# 프론트엔드 정적 파일 버전 관리 가이드

## 버전 규칙

JS, CSS 파일을 HTML에서 로드할 때 URL에 `?v=` 쿼리스트링을 붙여 브라우저 캐시를 무효화한다.

### 형식

```
?v=YYYYMMDD-N
```

- `YYYYMMDD` — 수정한 날짜
- `N` — 같은 날 수정 순번 (1부터 시작, 당일 추가 수정 시 증가)

### 예시

```html
<link rel="stylesheet" href="css/style.css?v=20260618-1">
<script src="js/views/charts-view.js?v=20260618-2"></script>
```

---

## 운영 규칙

1. **JS/CSS 파일을 수정할 때마다** 해당 파일을 참조하는 모든 HTML의 `?v=` 값을 함께 올린다.
2. **파일마다 버전을 독립적으로 관리**한다. 전역 버전이 아니므로, 수정한 파일만 올린다.
3. **같은 파일이 여러 HTML에 쓰이면** 모든 HTML에서 동일하게 버전을 올린다.
4. **같은 날 두 번째 수정**은 순번만 올린다. (`20260618-1` → `20260618-2`)
5. `?v=` 갱신 없이 내용만 바꾸면 사용자가 캐시된 구버전을 볼 수 있다. 반드시 함께 커밋한다.
6. **ES 모듈(ESM) 사용 시 주의**: `type="module"` 스크립트는 이 방식이 적용되지 않는다. HTML에서 로드하는 진입 스크립트의 `?v=`를 변경해도 그 내부에서 `import`하는 하위 모듈 파일들의 캐시는 무효화되지 않는다. ESM을 쓰는 프로젝트는 webpack/vite 등 번들러의 콘텐츠 해시(`[contenthash]`) 방식을 사용해야 한다.
